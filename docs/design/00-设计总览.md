# 00 - 设计总览

## 项目一句话

`License` 是厂商内部的私有化产品 License 签发与管理平台，签发主体是厂商，接收主体是部署在客户内网的产品实例，同时支持在线与离线激活。

## 核心目标

1. 让厂商可以安全签发 / 续期 / 撤销 License，全过程可审计。
2. 让产品实例在「无外网」也能完成激活与持续校验。
3. 让产品实例在「有外网」时，自动获得心跳、撤销与续期能力。
4. 让所有签发、激活、撤销、登录、配置变更行为都进入哈希链审计，且可被签名导出。
5. 让平台自身没有已知漏洞类、没有后门、没有调试旁路。

## 非目标（第一版明确不做）

- 不做按 feature 颗粒度签发；License 一次性开放完整功能。
- 不做多角色 RBAC，仅一个超级管理员。
- 不做客户自助门户；客户与厂商通过 Console + Activation Code 交互。
- 不做计费、订阅、续费的财务流程，仅做技术层面的过期与续期。
- 不做异步任务；当前不引入 worker 镜像。
- 不做对象存储；License 文件直接由 API 返回。

## 关键判断

- 安全是首位。任何与安全冲突的便利性优化必须被否决。
- 离线优先。所有验证逻辑必须能在无网络下完成。
- 离线撤销不可达是物理限制，不假装能解决，通过短有效期 + 必选在线检查（C3 可选）缓解。
- License 一次签发开放完整功能；越简单越可信。
- 第一版按 N 个产品设计、只接入 1 个产品；扩展零迁移。

## 验证方式

- 第一版交付时，必须能演示：在线签发、离线签发、在线撤销、离线到期、产品端纯本地校验通过。
- 第一版必须通过：bandit + semgrep + pip-audit + safety + npm audit 全部 0 高危。
- 第一版必须有：登录限速 + 锁定 + 审计哈希链篡改测试通过。
- 第一版必须能：从一个完整快照的 Postgres + Redis 复原 Console 状态且签发链不丢。

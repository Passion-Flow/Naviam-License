# 04 - 目录设计

本文件是 License 项目目录强约束。所有 AI 协作和人工开发都必须遵守。

## 顶层目录

```text
projects/license/
  .agent/             # 项目规则、上下文、会话记录
  apps/               # 应用入口（仅 api、web）
    api/              # Django 项目
    web/              # Next.js 项目
  src/                # 共享 Python 源码
    modules/          # 业务模块（Django app）
    shared/           # 通用工具
    contracts/        # 接口契约源文件
    config/           # 配置 helper
  sdk/                # 独立 pip 包 license_sdk
  deploy/             # Service 层 Docker Compose（仅 postgres、redis）
  infra/              # Helm chart
  tests/              # 集成 / e2e / 安全
  docs/               # 设计 / 安全 / api / 决策 / 部署
  services/           # 工作区骨架保留（第一版未启用）
  .env.example
  .gitignore
  .editorconfig
  README.md
  LICENSE
  CHANGELOG.md
```

## 强约束

### 1. 业务模块 = 目录

每个完整的业务能力对应一个目录。例：登录是一个目录，里面装登录所需的所有文件。

不允许：「一个按钮一个目录」「一个工具函数一个目录」「一个常量一个目录」。

正例：

```text
src/modules/accounts/                # 登录 / 2FA / 改密 / 锁定 / 会话整体
  models.py
  views.py
  services.py
  serializers.py
  urls.py
  permissions.py
  middleware.py
  tests/
```

反例：

```text
src/modules/accounts/login/button/   # 不允许
src/modules/accounts/login/form/     # 不允许
src/modules/accounts/login/handler/  # 不允许
```

### 2. ≥ 2 个有用文件才建目录

任何目录至少必须有 2 个有用文件。否则不要创建该目录，直接和兄弟项平级。

`__init__.py`、`.gitkeep`、`README.md` 不计入「有用文件」。

### 3. ≤ 4 层深度

从 `projects/license/` 起算，业务文件最大深度为 4。例：

```text
projects/license/src/modules/licenses/services.py   # 5 段，但 modules 与 services.py 是平级文件，深度 = 4 OK
projects/license/src/modules/licenses/x/y/z.py      # 深度 = 6，禁止
```

唯一例外：`apps/web/src/app/` 下的 Next.js App Router 路由树，因 Next.js 路由约定不可避免，允许：

```text
apps/web/src/app/(dashboard)/licenses/[id]/page.tsx
apps/web/src/app/(dashboard)/licenses/[id]/revoke/page.tsx
```

且仅允许「路由 segment」目录嵌套；非路由目录仍受 4 层限制。

### 4. 单文件组件不建目录

如果一个组件只有一个文件，直接放到兄弟组件平级，不建目录。

正例：

```text
apps/web/src/components/license-card.tsx
apps/web/src/components/cloud-id-input.tsx
apps/web/src/components/activation-code-display.tsx
```

反例：

```text
apps/web/src/components/license-card/index.tsx      # 仅一个文件，禁止建目录
apps/web/src/components/cloud-id-input/CloudIdInput.tsx  # 同上
```

如果该组件需要拆分为 ≥ 2 个文件（主组件、样式、子组件），可以建目录，且目录内必须 ≥ 2 个有用文件：

```text
apps/web/src/components/license-issue-form/
  index.tsx
  fields.tsx
  schema.ts
```

### 5. 单文件 ≈ 400 行触发拆分

单文件超过约 400 行时必须拆分。低于 400 行不要为「以后可能扩展」预先拆分。

拆分顺序：

1. 同目录下按职责拆 2~3 个文件。
2. 仍然臃肿，建目录（必须 ≥ 2 文件）。
3. 涉及多模块共用，提到 `src/shared/` 或 `apps/web/src/lib/`。

## 模块标准骨架

### Django 业务模块

```text
src/modules/<module>/
  __init__.py
  apps.py
  models.py
  serializers.py
  views.py
  services.py
  selectors.py        # 可选；只在跨模型只读查询多到 services.py 臃肿时才建
  urls.py
  admin.py
  permissions.py      # 可选；按需
  middleware.py       # 可选；按需
  migrations/
  tests/
  README.md           # 模块边界 / 职责 / 上下游依赖
```

### Next.js 路由（dashboard 子区）

```text
apps/web/src/app/(dashboard)/licenses/
  page.tsx                  # 列表
  layout.tsx                # 子区布局（可选）
  [id]/
    page.tsx                # 详情
    revoke/
      page.tsx              # 撤销表单
```

### SDK 平铺

```text
sdk/src/license_sdk/
  __init__.py
  client.py
  loader.py
  validator.py
  crypto.py
  online.py
  errors.py
```

不为 `client.py` 建 `client/__init__.py`、不为 `crypto.py` 建 `crypto/utils.py`。

## 前后端模块对称

| 业务能力 | 前端路由 / 组件目录 | 后端模块 | 数据库主表 |
|---|---|---|---|
| 账户与登录 | `apps/web/src/app/(auth)/login/` + `components/auth-*` | `src/modules/accounts/` | `accounts_user`、`accounts_login_attempt` |
| 客户管理 | `apps/web/src/app/(dashboard)/customers/` | `src/modules/customers/` | `customers_customer` |
| 产品管理 | `apps/web/src/app/(dashboard)/products/` | `src/modules/products/` | `products_product` |
| License 主流程 | `apps/web/src/app/(dashboard)/licenses/` | `src/modules/licenses/` | `licenses_license` |
| 激活与心跳 | `apps/web/src/app/(dashboard)/licenses/[id]/activate/` | `src/modules/activations/` | `activations_cloud_id`、`activations_heartbeat` |
| 审计 | `apps/web/src/app/(dashboard)/audit/` | `src/modules/audit/` | `audit_event` |
| 通知 | `apps/web/src/app/(dashboard)/notifications/` | `src/modules/notifications/` | `notifications_channel`、`notifications_outbox` |
| 安全配置 | `apps/web/src/app/(dashboard)/settings/security/` | `src/modules/security/` | 无业务表，只有运行期资源 |

## 调整记录格式

任何对本文件目录约束的修改都必须遵循：

```text
日期：
新增或调整的目录约束：
原因：
影响范围：
是否同步全局：
```

如果某条约束被多个项目验证有效，按 `.agent/rules/` 流程同步到工作区全局规则。

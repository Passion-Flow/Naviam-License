# 08 - 服务层设计

## 第一版启用

| 类别 | Service | 用途 |
|---|---|---|
| 数据库 | Postgres 16 | 主数据：账户、客户、产品、License、激活、审计、通知 |
| 缓存 | Redis 7 | Session、限速计数、登录锁定计数、心跳缓冲、轻量队列（`rq` / `dramatiq` 后续可选） |

详见：

- `deploy/database/postgres/`
- `deploy/cache/redis/`

启动方式：

```bash
cd projects/license/deploy/database/postgres
docker compose up -d

cd projects/license/deploy/cache/redis
docker compose up -d
```

应用代码不通过 Docker 启动；按本机 Python / Node 直接运行（详见 `apps/api/README.md` 与 `apps/web/README.md`）。

## 暂不引入

| Service | 原因 | 触发条件 |
|---|---|---|
| MinIO | 第一版无文件上传；License 文件直接由 API 返回 | 合规导出需要大附件 / 多媒体附件 |
| Elasticsearch | 第一版无搜索 / 向量需求 | 审计跨年检索吞吐瓶颈 |
| Nginx Service | 私有化部署阶段补反代；本地直连即可 | Helm/Docker 私有化交付正式启动 |
| RabbitMQ / Kafka | 第一版无异步任务 | 心跳吞吐爆增 / 引入 worker 镜像 |
| ClickHouse / OLAP | 第一版无分析需求 | 客户用量分析 / 合规报表 |
| Vector DB | 与本项目无关 | 不会引入 |

## Postgres 配置原则

- 仅在 service network 暴露端口；生产 Helm 不暴露 hostport。
- 用户名/库名变量化：`{PROJECT_SLUG}_postgres` -> `license_postgres`。
- 密码由 `.env` 注入，不在镜像内或 git 中保存默认值。
- 启用 `scram-sha-256` 认证。
- 启用 `pgaudit`（生产）。
- 备份：每日 `pg_basebackup` + WAL 归档；恢复演练每月一次。

## Redis 配置原则

- `requirepass` 通过 `redis.conf.template` 注入，不走 CLI 参数（避免历史泄漏）。
- 启用 `maxmemory` + `maxmemory-policy allkeys-lru`，防止内存爆。
- `bind` 限定到 service network。
- 不启用持久化的危险组合（仅 RDB 不够时启用 AOF + 每秒 fsync）。
- 不允许 `CONFIG GET *` 远程获取；运维通过容器内执行。

## 端口与网络

- Postgres：服务网络内部 `postgres:5432`。
- Redis：服务网络内部 `redis:6379`。
- API：本机 `127.0.0.1:8080`（开发）。
- Web：本机 `127.0.0.1:3000`（开发）。
- 私有化部署：Helm chart 中 API + Web 都通过 ClusterIP，对外通过 Ingress + TLS。

## 数据生命周期

- License、审计、签发记录：永久保留（合规需要）。
- 心跳：仅保留 30 天明细 + 月度汇总。
- 登录失败 / 锁定记录：保留 90 天。
- Session：默认 1 小时滑动过期，最大 12 小时；管理员可一键全部失效。

## 备份与恢复

- 每日 Postgres `pg_basebackup` + WAL；离线归档到客户合规存储。
- Redis 仅作为缓存；丢失不影响业务正确性，但限速计数会重置。
- Console 状态完全可从 Postgres 恢复；签发链不依赖 Redis。

## 灾难恢复演练

- 每季度一次：从备份还原 Postgres、重启 API、Console 进入只读但仍能签发紧急 License、审计链 hash 校验通过。
- 演练记录写入 `docs/decisions/` 下的 ADR。

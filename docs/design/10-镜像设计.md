# 10 - 镜像设计

## 第一版镜像

| 镜像 | 路径 | 受众 | 暴露面 |
|---|---|---|---|
| `license-api` | `apps/api/Dockerfile` | 厂商 Console 后端 | 仅内网 / 厂商内部 |
| `license-web` | `apps/web/Dockerfile` | 厂商管理员浏览器 | 仅内网 / 厂商内部 |

第一版仅这两个。其他镜像全部暂不引入。

## 暂不引入

| 镜像 | 原因 | 触发条件 |
|---|---|---|
| `worker` | 第一版无异步任务 | 心跳吞吐 / 通知队列 / 大批量过期任务 |
| `gateway` | 第一版直连 API；私有化 Helm 由 Ingress 承担反代 | 多租户 / 企业版上线 |
| `sandbox` | 与本项目无关 | 不引入 |
| `enterprise-web` | 受众与本 Console 重合 | 客户自助门户上线 |
| `enterprise-worker` | 同上 | 同上 |
| 单独的 `web`（终端用户） | 本项目仅厂商内部使用 | 客户自助门户上线 |

## 镜像基底

- 后端：`gcr.io/distroless/python3-debian12:nonroot`（生产）+ 中间构建用 `python:3.12-slim`。
- 前端：`gcr.io/distroless/nodejs20-debian12:nonroot`（生产）+ 中间构建用 `node:20-bookworm-slim`。
- 仅运行非 root 用户。
- 只读根文件系统 + 显式可写卷。

## 多阶段构建

```text
后端:
  builder (python:3.12-slim) -> 安装依赖（hash-locked） -> 收集 collectstatic -> 拷贝 wheels
  runtime (distroless) -> 复制依赖 + 应用代码

前端:
  deps (node:20-slim) -> pnpm install --frozen-lockfile
  builder -> pnpm build
  runtime (distroless nodejs) -> 复制 .next/standalone
```

## 镜像签名 / SBOM / 扫描

- 构建后 `cosign sign --keyless`（OIDC）或离线签名（运行 GPG keypair）。
- `syft packages dir:.` 生成 SBOM；附在 release。
- `trivy image --severity HIGH,CRITICAL` 扫描；高危 0 通过门禁。

## 多架构

- `linux/amd64` + `linux/arm64`。
- `docker buildx build --platform linux/amd64,linux/arm64`。
- ARM Mac 开发者本地可直接拉对应架构。

## 离线交付

- 离线交付包：镜像 tar (`docker save`) + Helm chart + values 模板 + SBOM + 签名 + README。
- 客户内网无法访问公共仓库时，使用客户私有仓库 / 本地 registry。
- 镜像版本：语义化 + 构建 metadata（commit short sha + build time）。

## 健康检查

- API：`GET /healthz` -> 200；`GET /readyz` 检查 Postgres + Redis + 私钥可加载。
- Web：`GET /api/health` -> 200。
- Helm liveness / readiness 配置使用上述端点。

## 不主动构建

第一版不主动 build 镜像。只在用户明确要求或 CI 触发时构建。开发期 API 与 Web 都本机直接运行（`python manage.py runserver` / `pnpm dev`）。

# 11 - 企业私有化部署约束

本项目的 Console 既部署在厂商内部，也作为厂商交付能力的一部分。SDK 嵌入到客户产品镜像。

## 核心约束

- 配置变量化：所有可变项进入 `.env.example`；生产环境必须覆盖默认账号、密钥、数据库密码。
- Service 层独立 Compose；应用代码默认手动启动。
- 默认管理员 `Admin / admin@workerspace.ai / admin@workerspace.ai` 必须在首次登录强制改密 + 绑定 2FA，否则平台拒绝继续操作。
- 不允许调试旁路、隐藏管理员、未审计的高权限端点。
- 不允许将密钥、passphrase、token 写入镜像、代码、git。

## 交付形态

- 厂商 Console：Helm chart（首选）+ Docker Compose（备选）。
- 客户产品：Python pip 包 `license_sdk`，由产品自行嵌入并构建到客户镜像。

## 多架构 / 多 OS

- 镜像同时构建 `linux/amd64` 与 `linux/arm64`。
- 浏览器：Chrome / Edge / Firefox / Safari 当代版本（厂商管理员桌面）。
- 桌面：Mac / Linux / Windows 浏览器均必须可用。

## 网络环境

- 国内：默认走国内仓库源；生产由客户配置内网代理。
- 国外：基础镜像与 Python / Node 依赖镜像源可切换。
- 离线：所有依赖可通过离线包注入；构建过程不依赖外网。

## TLS

- 生产强制 TLS 1.3；HSTS preload；HTTP 自动 301 到 HTTPS。
- Console 与产品实例之间使用 mTLS：双方互验公钥。
- Helm chart 内置 cert-manager 集成（可选）；客户自带证书也可。

## 备份与恢复

- Postgres 每日 `pg_basebackup`，WAL 归档。
- 签发私钥：客户运维负责保管 passphrase；推荐冷存与轮换。
- 恢复演练：每季度一次，过程写入审计。

## 监控

- Prometheus `/metrics`：限速触发次数、登录失败、签发次数、撤销次数、心跳延迟、错误率。
- Grafana 模板：随 Helm chart 提供。
- Alertmanager 触发条件：登录失败激增、签发激增、撤销激增、私钥不可用、Postgres / Redis 不可用。

## 合规

- 审计哈希链可签名导出（公钥 + 链 hash + 数据），便于客户合规审查。
- 默认 Asia/Shanghai；客户域内时区可调。
- 数据保留期写入设置：License、签发记录、审计永久；心跳明细 30 天 + 月汇总。

## 升级

- Console 升级流程：备份 -> 滚动升级 API + Web -> 校验健康 -> 解除维护。
- 数据库迁移：仅向后兼容；任何破坏性变更必须经过 ADR 与影子库验证。
- 公钥替换：通过 `signature_kid` 多代号同时有效；新签发使用新 kid，旧 License 仍可验签。

## 主题与窗口

- 明 / 暗 / 跟随系统三档；用户偏好持久化。
- 自适应窗口：≥ 1280 桌面优化；移动端不是第一版重点但布局必须不错位。

## 私有化清单（交付时核对）

- 全部默认密码已改。
- 私钥已通过 A 方案加密、passphrase 由客户运维设定。
- 备份与还原演练已通过。
- 公钥已注入 SDK 并验签产物可在客户产品离线运行。
- TLS 证书与域名已就位。
- 审计导出格式已约定。

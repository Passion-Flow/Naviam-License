
## 公共契约

接口源文件位于 `src/contracts/`：

```text
src/contracts/
  __init__.py
  errors.py            # 全站统一错误码
  pagination.py        # 列表分页协议
  audit.py             # 审计写入接口
  signing.py           # IKeySigner 抽象（A 方案 / KMS / HSM 共用）
```

所有 DRF 视图直接消费 `src/contracts/`，避免每个模块自创错误码。

## 状态机

License 状态：

```text
draft -> issued -> active -> {expired, revoked}
                              expired -> grace -> sunset
                              revoked -> sunset
```

详细：

- `draft`：Console 表单填好但尚未签名。
- `issued`：已签名、未送达。
- `active`：已落地，product 端校验通过。
- `expired`：到期，进入只读。
- `grace`：到期 + 30 天宽限。
- `sunset`：宽限期结束 / 撤销结束，业务功能停止。
- `revoked`：已撤销。

状态迁移由 `src/modules/licenses/services.py` 集中处理；任何旁路修改 status 必须经过 `Service` 函数 + 写审计。

## 视图风格

- 所有视图基于 DRF `APIView` / `ViewSet`，禁止使用裸 `@api_view` 简化版。
- 显式 `permission_classes`、`throttle_classes`、`authentication_classes`。
- 列表视图统一通过 `pagination.LicensePagination`。
- 写操作必须经过 `services.<action>`，视图层只做 I/O。

## 错误处理

- 全站使用统一 `LicenseAPIError(code, message, http_status, hint)`。
- DRF `EXCEPTION_HANDLER` 把所有未捕获异常映射成 5xx + traceid，并写审计。
- 不在响应里泄漏内部错误。

## 限速

- 登录端点：5 次/分钟/IP + 5 次/分钟/账号；超过即锁定（`django-axes`）。
- 签发端点：10 次/分钟/账号；防止误操作和暴力签发。
- 心跳端点：60 次/分钟/实例 + IP；超过即拒绝并审计。
- 默认全站 60 次/分钟/IP。

## 会话与 CSRF

- Session 后端：`django.contrib.sessions.backends.signed_cookies` 不使用；统一 Redis 后端。
- `SESSION_COOKIE_SAMESITE = "Strict"`、`HttpOnly`、`Secure`。
- 所有写操作要求 CSRF Token；使用 Django 内置 `CsrfViewMiddleware` + 双提交 cookie + header。

## 审计

每个 service 函数在提交前调用 `audit.append(event)`：

```text
event = {
  prev_hash, ts, actor_id, actor_ip, action, target, payload, request_id
}
record_hash = sha256(prev_hash || canonical(event))
server_signature = ed25519.sign(audit_key, record_hash)
```

`audit_event` 表存 `(id, prev_hash, hash, signature, ...)`。任何记录被改动，链下 `record_hash` 与上一条 `prev_hash` 失配立即可证伪。

## 密钥管理（IKeySigner）

```text
class IKeySigner(Protocol):
    def public_key(self) -> bytes: ...
    def sign(self, payload: bytes) -> bytes: ...
```

实现：

- `FileKeySigner`：A 方案。`age` / `sops` 加密文件 + 启动时通过 passphrase 解密到内存，进程级单例。
- `KmsKeySigner`：预留接口，第二阶段实现。
- `HsmKeySigner`：预留接口，第三阶段实现。

私钥永远不出进程；签名永远不在 worker / 客户端进行。

## 部署相关

- `wsgi.py` / `asgi.py` 都提供；生产用 `gunicorn` + `uvicorn[standard]`。
- 健康检查：`/healthz`、`/readyz`，`/readyz` 检查 Postgres + Redis + 私钥可加载。
- `manage.py migrate` 在容器启动前执行；失败即拒启动。

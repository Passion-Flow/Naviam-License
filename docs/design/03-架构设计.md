# 03 - 架构设计

## 顶层架构

```text
+---------------------------+        +----------------------------+
|  厂商 Console (Next.js)   |  HTTPS |  Console API (Django/DRF)  |
|  apps/web                 +<------>+  apps/api                  |
+-------------+-------------+        +------+----------+----------+
              |                             |          |
              | (Cookie Session)            |          |
              |                             |          |
              v                             v          v
        +------------+                 +---------+ +-------+
        |  浏览器     |                 | Postgres| | Redis |
        +------------+                 +---------+ +-------+

                                                   ^
                                                   | (限速 / 锁定 / 心跳)
                                                   |
+----------------------+   离线: Cloud ID/Activation Code    +-------------------------+
|  客户产品实例         |   在线: HTTPS + 互信公钥             |  Console API            |
|  (内置 license_sdk)   +<------------------------------------>+  /v1/activations/...    |
+----------------------+                                       +-------------------------+
```

## 分层

```text
apps/                # 入口（仅 Web 与 API 镜像）
src/modules/         # 业务模块（Django app）
src/shared/          # 通用工具（dto、异常、序列化）
src/contracts/       # 接口契约（schema 源文件）
src/config/          # 全局配置 helper
sdk/                 # 独立 pip 包
deploy/              # Service 层 Compose
infra/helm/license/  # Helm chart
docs/                # 设计 / 安全 / api / 决策 / 部署
tests/               # 集成、e2e、安全
```

模块（`src/modules/`）：

```text
accounts        登录、2FA、会话、改密、锁定
customers       客户档案
products        产品定义（第一版仅 1 条记录）
licenses        License 签发 / 续期 / 撤销主流程
activations     Cloud ID 解码 / Activation Code 编码 / 心跳
audit           审计哈希链
notifications   通知（邮件 / Webhook）
security        密钥管理 / 限速 / 头部 / 审计签名 / secrets
```

## 信任边界

| 边界 | 内 | 外 | 控制 |
|---|---|---|---|
| 浏览器 ↔ Web | 用户输入 | Web 后端 | CSRF、CSP、Cookie SameSite |
| Web ↔ API | 已认证会话 | 公网 | Session + IP 白名单（可选）+ 限速 |
| API ↔ Postgres/Redis | 服务网络 | API | 仅服务网络可达；密码 + TLS（生产）|
| 厂商 Console ↔ 产品实例 | 互信 | 客户内网 | mTLS + 双向公钥校验 |
| SDK ↔ License 文件 | 进程内 | 文件系统 | 公钥验签 + 完整性校验 |

## 默认拒绝

任何不能通过策略明示放行的请求一律拒绝；放行必须显式：

- 视图层：`permission_classes` 明确写出。
- 网络层：仅放行需要的端口与 host。
- 数据层：行级访问控制基于 `customer_id`、`product_id` 显式过滤。
- 文件层：除签发产物外，不接收任何上传。

## 失败模式

- 数据库不可达：API 立即返回 5xx，Console 显示「服务不可用」，不降级到内存模拟。
- Redis 不可达：登录锁定 / 限速降级到 fail-closed（拒绝），不放行。
- 私钥不可加载：API 拒启动，输出错误日志（不打印密钥本身或 passphrase）。
- 心跳不可达（产品侧）：仅记录本地日志，进入容忍期；不主动失效。

## 关键时序

### 在线签发

```text
管理员 -> Console -> POST /v1/licenses/issue
                  -> 创建 License + 哈希链审计
                  -> Ed25519 私钥签名
                  -> POST 客户实例激活回调（mTLS）
                  -> 客户实例落地公钥验签
                  -> 回执
                  -> 审计哈希链追加「已落地」
```

### 离线签发

```text
客户 UI 复制 Cloud ID
管理员粘贴 -> POST /v1/licenses/issue { cloud_id, customer_id, expires_at }
            -> 解码 Cloud ID -> 校验 product_id + 公钥指纹 + 硬件指纹
            -> 创建 License + 哈希链审计
            -> Ed25519 私钥签名
            -> 输出 Activation Code (Base32 + 校验位)
管理员复制 Activation Code 给客户
客户 UI 粘贴 -> SDK 公钥验签 -> 落地 -> 进程内有效
```

### 撤销

```text
管理员 -> Console -> POST /v1/licenses/{id}/revoke
                  -> 标记 status=revoked + 哈希链审计
                  -> 在线: POST 客户实例撤销回调
                            -> 客户实例下次心跳/启动确认撤销
                  -> 离线: 通知 Webhook + 邮件 + 标记「等待自然到期」
```

## 可观测性

- 应用日志：JSON 格式 + 关键字段（trace_id、actor、action、target、result）。
- 审计：单独表 + 哈希链；导出可签名压缩包。
- 健康检查：`/healthz`（liveness）+ `/readyz`（依赖通联）。
- Metrics：`/metrics`（Prometheus），仅在内网可达。

## 扩展方向（不进入第一版）

- 多产品上线后，按 `product_id` 横向扩展；不需迁移。
- 多角色 RBAC：在 `accounts` 模块按 ADR 启用。
- KMS / HSM：实现 `IKeySigner`，替换文件后端。
- 客户自助门户：单独 `apps/customer-web`，独立网络域。

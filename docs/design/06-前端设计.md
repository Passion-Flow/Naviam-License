# 06 - 前端设计

## 项目骨架

```text
apps/web/
  package.json
  next.config.js
  tsconfig.json
  tailwind.config.ts
  postcss.config.js
  src/
    app/
      layout.tsx
      page.tsx
      (auth)/
        login/
          page.tsx
      (dashboard)/
        layout.tsx
        licenses/
          page.tsx
          [id]/
            page.tsx
        customers/
          page.tsx
        products/
          page.tsx
        audit/
          page.tsx
        notifications/
          page.tsx
        settings/
          page.tsx
      api/                # 仅给 Next.js 自身使用的轻量路由（如 health）
    components/
    lib/
    hooks/
    types/
    styles/
```

## 路由

- `(auth)`：未登录可访问，仅 `login` 一项。
- `(dashboard)`：已登录可访问，所有业务页面在此。
- 中间件 `middleware.ts` 在请求层判断 Cookie 会话，未登录强制跳 `/login`。

## 数据流

- 客户端：`react-query` (TanStack Query) 管理远端状态。
- 表单：`react-hook-form` + `zod`，校验规则与后端契约同源。
- 全局状态：避免引入额外 store；用 react-query + URL state。
- API 客户端：`apps/web/src/lib/api-client.ts`，使用 `fetch` + 自动附带 CSRF token + cookie，超时 + 重试有限。

## 契约

前后端共享契约位于 `src/contracts/`（Python）+ `apps/web/src/types/contracts.ts`（TypeScript 镜像）。

变更流程：

1. 修改 `src/contracts/`。
2. 通过 `/sync-api-contract` 命令生成 TS 类型并更新 `apps/web/src/types/contracts.ts`。
3. 更新 `docs/api/contracts.md`。
4. 同步前端调用点。

## 安全策略

- 不在 `localStorage` 存任何敏感数据（包括用户名 / 邮箱 / token）。
- 敏感数据只走 HttpOnly Cookie。
- 表单提交统一带 CSRF token（来自 cookie + `X-CSRFToken` header）。
- 错误信息不暴露后端内部细节，仅显示统一错误码 + 简短描述 + traceid（便于联调）。
- 所有外链必须 `rel="noopener noreferrer"` + 显式协议白名单。

## CSP

`next.config.js` 输出严格 CSP：

```text
default-src 'self';
script-src 'self';
style-src 'self' 'unsafe-inline';     # tailwind 注入；可改为 hash-based
img-src 'self' data:;
connect-src 'self' https://<console-api-host>;
frame-ancestors 'none';
form-action 'self';
base-uri 'self';
```

## 主题

- 明暗主题：`shadcn/ui` 内置；`html data-theme` 切换。
- 系统跟随 + 用户偏好 + 持久化（cookie，不是 localStorage）。
- 自适应窗口：移动端不是第一版重点，但布局必须优雅退化（不可错位）。

## 国际化

- 第一版仅中文。架构按 i18n-ready：所有文本走 `t('...')`，方便后期切英文。
- 时区显示：默认 `Asia/Shanghai`；用户可在 `settings` 切换。

## 性能

- 首屏：Next.js 默认按需 chunk + Tailwind purge。
- 路由切换：`Link prefetch` 仅在视口内启用。
- 表格：审计列表按虚拟滚动（`@tanstack/react-virtual`），避免渲染万级数据。

## 可访问性

- 所有交互组件来自 `shadcn/ui`，已具备 ARIA 支持。
- 自定义组件必须保留键盘可达性、aria-label、focus-visible。

## 测试

- 单元：`vitest` + `@testing-library/react`。
- E2E：`playwright`，覆盖登录 / 2FA / 签发 / 撤销 / Cloud ID 粘贴 / Activation Code 复制 / 错误页。

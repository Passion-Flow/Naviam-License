# 09 - 数据库设计 - 库表字段

数据库：Postgres 16；schema：`public`；约定全部表名前缀按 Django 默认规则 `<app_label>_<model>`。

时间字段一律 `timestamptz`，写入存 UTC，展示按用户时区。

UUID 主键统一使用 `uuid` 类型，默认 `gen_random_uuid()`（启用 `pgcrypto` 扩展）。

## 模型清单

### accounts

```sql
accounts_user
  id                uuid PK
  username          citext UNIQUE NOT NULL
  email             citext UNIQUE NOT NULL
  password          varchar(255) NOT NULL          -- Argon2id
  is_active         boolean NOT NULL DEFAULT true
  is_superadmin     boolean NOT NULL DEFAULT false
  must_change_pw    boolean NOT NULL DEFAULT true  -- 首次登录强制改密
  totp_secret       bytea NULL                     -- 加密存储
  totp_confirmed    boolean NOT NULL DEFAULT false
  recovery_codes    bytea NULL                     -- 加密 JSON
  last_login_at     timestamptz NULL
  last_login_ip     inet NULL
  created_at        timestamptz NOT NULL DEFAULT now()
  updated_at        timestamptz NOT NULL

accounts_login_attempt
  id                bigserial PK
  username          citext NOT NULL
  ip                inet NOT NULL
  ua                text NULL
  result            varchar(16) NOT NULL          -- success / bad_password / locked / 2fa_failed
  reason            text NULL
  created_at        timestamptz NOT NULL DEFAULT now()
  INDEX (username, created_at DESC)
  INDEX (ip, created_at DESC)
```

### customers

```sql
customers_customer
  id                uuid PK
  display_name      varchar(255) NOT NULL
  legal_name        varchar(255) NULL
  contact_name      varchar(128) NULL
  contact_email     citext NULL
  contact_phone     varchar(64) NULL
  region            varchar(64) NULL
  notes             text NULL
  created_at        timestamptz NOT NULL DEFAULT now()
  updated_at        timestamptz NOT NULL
  UNIQUE (display_name)
```

### products

```sql
products_product
  id                uuid PK
  code              varchar(64) NOT NULL UNIQUE   -- 第一版仅一条种子 'default'
  display_name      varchar(255) NOT NULL
  description       text NULL
  schema_version    int NOT NULL DEFAULT 1        -- Cloud ID / Activation Code 协议版本
  created_at        timestamptz NOT NULL DEFAULT now()
  updated_at        timestamptz NOT NULL
```

### licenses

```sql
licenses_license
  id                uuid PK
  license_id        varchar(64) NOT NULL UNIQUE   -- 业务 ID，对外暴露
  product_id        uuid NOT NULL REFERENCES products_product(id)
  customer_id       uuid NOT NULL REFERENCES customers_customer(id)
  cloud_id_binding  bytea NOT NULL                -- 解码后的二进制
  cloud_id_text     text NOT NULL                 -- 原始 Cloud ID
  hardware_fp_hash  bytea NOT NULL
  instance_pubkey   bytea NOT NULL
  status            varchar(16) NOT NULL          -- draft|issued|active|expired|grace|sunset|revoked
  issued_at         timestamptz NULL
  not_before        timestamptz NULL
  expires_at        timestamptz NOT NULL
  grace_until       timestamptz NULL
  signature         bytea NULL                    -- Ed25519
  signature_algo    varchar(32) NOT NULL DEFAULT 'ed25519'
  signature_kid     varchar(64) NOT NULL          -- 签名密钥 ID
  payload_canonical bytea NOT NULL                -- 用于验签的规范化字节
  notes             text NULL
  issued_by         uuid NOT NULL REFERENCES accounts_user(id)
  created_at        timestamptz NOT NULL DEFAULT now()
  updated_at        timestamptz NOT NULL
  INDEX (customer_id, status)
  INDEX (product_id, status)
  INDEX (expires_at)
```

### activations

```sql
activations_cloud_id
  id                uuid PK
  license_id        uuid NULL REFERENCES licenses_license(id)
  product_code      varchar(64) NOT NULL
  instance_id       varchar(128) NOT NULL
  pubkey_fp         varchar(128) NOT NULL
  hardware_fp       varchar(128) NOT NULL
  schema_version    int NOT NULL
  raw_text          text NOT NULL
  created_at        timestamptz NOT NULL DEFAULT now()
  UNIQUE (product_code, instance_id, hardware_fp)

activations_heartbeat
  id                bigserial PK
  license_id        uuid NOT NULL REFERENCES licenses_license(id)
  reported_at       timestamptz NOT NULL
  client_ip         inet NULL
  client_version    varchar(64) NULL
  status            varchar(16) NOT NULL          -- ok / mismatch / expired / revoked
  detail            jsonb NULL
  INDEX (license_id, reported_at DESC)
```

### audit

```sql
audit_event
  id                bigserial PK
  prev_hash         bytea NOT NULL
  hash              bytea NOT NULL
  signature         bytea NOT NULL                -- Ed25519
  signature_kid     varchar(64) NOT NULL
  ts                timestamptz NOT NULL DEFAULT now()
  actor_id          uuid NULL REFERENCES accounts_user(id)
  actor_kind        varchar(32) NOT NULL          -- user / system / sdk
  actor_ip          inet NULL
  action            varchar(64) NOT NULL          -- license.issue / license.revoke / login.* / settings.*
  target_kind       varchar(32) NULL
  target_id         varchar(128) NULL
  request_id        varchar(64) NULL
  payload           jsonb NOT NULL
  UNIQUE (hash)
  INDEX (ts DESC)
  INDEX (action, ts DESC)
```

### notifications

```sql
notifications_channel
  id                uuid PK
  customer_id       uuid NULL REFERENCES customers_customer(id)
  kind              varchar(32) NOT NULL          -- email / webhook
  destination       text NOT NULL                 -- 邮件地址 / URL
  secret            bytea NULL                    -- Webhook 共享密钥（HMAC-SHA256）
  is_active         boolean NOT NULL DEFAULT true
  created_at        timestamptz NOT NULL DEFAULT now()
  updated_at        timestamptz NOT NULL

notifications_outbox
  id                bigserial PK
  channel_id        uuid NOT NULL REFERENCES notifications_channel(id)
  topic             varchar(64) NOT NULL          -- license.revoked / license.expiring / ...
  payload           jsonb NOT NULL
  status            varchar(16) NOT NULL          -- queued / sent / failed
  attempt           int NOT NULL DEFAULT 0
  next_attempt_at   timestamptz NULL
  last_error        text NULL
  created_at        timestamptz NOT NULL DEFAULT now()
  INDEX (status, next_attempt_at)
```

## 迁移策略

- 工具：Django migrations。
- 流程：
  1. 修改 model。
  2. `python manage.py makemigrations`。
  3. `python manage.py migrate`（dev）。
  4. 预生产 / 生产先在影子库执行 `--plan`。
  5. 大表添加索引使用 `CREATE INDEX CONCURRENTLY`（手写 RunSQL）。
- 回滚：每个迁移必须可向后回滚（不删数据，必要时分两步）。
- 多环境同步：迁移文件提交进 git，CI 校验 `makemigrations --check --dry-run`。

## 初始化数据

`apps/api/config/fixtures/initial.json`：

- 默认管理员（来自 `.env`）。
- 默认产品 `code='default'`。
- 默认审计哈希链起点（genesis 记录 `prev_hash = b'\x00' * 32`）。

## 索引与查询模式

- License 列表按 `customer_id`、`status`、`expires_at` 高频查询。
- 审计按时间倒序、按 action 过滤。
- 心跳按 `license_id, reported_at DESC`，加 `BRIN` 索引压缩历史。

## 行级安全

- 第一版仅一个超级管理员，不启用 RLS。
- 多角色 / 多租户引入时按 ADR 启用。

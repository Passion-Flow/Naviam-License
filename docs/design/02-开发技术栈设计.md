# 02 - 开发技术栈设计

## 后端

| 候选 | 推荐 | 理由 |
|---|---|---|
| Django 5 + DRF | ✅ | 安全默认值最完善、ORM/迁移成熟、生态强、与既有 Python 产品语言一致 |
| FastAPI | ❌（第一版） | 安全默认值少、需要自行拼装 csrf/session/auth/admin |
| Go (gin/echo) | ❌ | 团队语言一致性优先 Python；安全默认值不如 Django |

最终方案：**Python 3.12 + Django 5 + Django REST Framework**。

### 关键库

| 用途 | 库 | 备注 |
|---|---|---|
| 数据库驱动 | `psycopg[binary,pool] >= 3.2` | 官方 v3 |
| 密码哈希 | `argon2-cffi` | Argon2id；Django 内置支持 |
| 2FA | `django-otp` + `pyotp` | TOTP；备用恢复码 |
| 登录锁定 | `django-axes` | 失败次数 + IP + 用户维度 |
| 限速 | `django-ratelimit` | 视图级与登录端点重点防护 |
| CSP / 安全头 | `django-csp` + Django 内置 `SecurityMiddleware` | HSTS、X-Frame、X-Content-Type、Referrer-Policy |
| 加密 | `cryptography` | Ed25519 / X25519 / HKDF；不自研 |
| 审计 | 自研模块（哈希链） | `prev_hash + payload + server_signature` |
| 任务调度 | 第一版不引入 | worker 镜像未启用 |
| 缓存 / 限速后端 | `redis-py` + Django Redis cache | 与限速、会话共用 |

### Django 关键安全配置（在 `apps/api/config/settings/base.py` 落地）

- `SESSION_COOKIE_SECURE = True`
- `SESSION_COOKIE_HTTPONLY = True`
- `SESSION_COOKIE_SAMESITE = "Strict"`
- `CSRF_COOKIE_SECURE = True`
- `CSRF_COOKIE_HTTPONLY = True`
- `SECURE_HSTS_SECONDS >= 31536000`
- `SECURE_HSTS_INCLUDE_SUBDOMAINS = True`
- `SECURE_HSTS_PRELOAD = True`
- `SECURE_SSL_REDIRECT = True`（生产）
- `SECURE_REFERRER_POLICY = "same-origin"`
- `SECURE_CROSS_ORIGIN_OPENER_POLICY = "same-origin"`
- `X_FRAME_OPTIONS = "DENY"`
- `PASSWORD_HASHERS` 首选 Argon2id
- `AUTH_PASSWORD_VALIDATORS` 强制最小复杂度与黑名单
- `DATA_UPLOAD_MAX_MEMORY_SIZE` / `FILE_UPLOAD_MAX_MEMORY_SIZE` 收紧
- `ALLOWED_HOSTS` / `CSRF_TRUSTED_ORIGINS` 显式
- `LANGUAGE_CODE = "zh-hans"`，`TIME_ZONE = "Asia/Shanghai"`

## 前端

| 候选 | 推荐 | 理由 |
|---|---|---|
| Next.js 14 App Router + TS | ✅ | 私有化 Console 主流、SSR 与流式可选、生态强、Tailwind/shadcn 配套 |
| Vite + React Router | ❌ | 缺路由约定、需要自行拼装权限拦截 |
| Vue 3 + Nuxt | ❌ | 团队若以 React 主导，统一栈 |

最终：**Next.js 14 App Router + TypeScript + shadcn/ui + Tailwind + React Hook Form + Zod**。

第一版以 SPA + 服务端 API 协作模式为主，不依赖 Next.js 服务端渲染做敏感数据。

## SDK

- 独立 pip 包 `license_sdk`。
- 无网依赖：仅依赖 `cryptography`。
- 暴露最小 API：`load_license(path) -> License`、`verify(license, cloud_id) -> Result`、可选 `online_check(...)`。
- 内置厂商公钥（构建时注入）。

## 数据库 / 缓存

| 选择 | 理由 |
|---|---|
| Postgres 16 | License、客户、审计哈希链、会话辅助 |
| Redis 7 | 限速计数器、登录锁定、Session 备份（如启用 Redis Session）、心跳缓冲 |

不启用：MinIO（License 文件直接 API 返回）、Elasticsearch（无搜索/向量需求）、Nginx Service（私有化阶段补反代）。

## 构建与部署

- 本地：`uv venv` + `uv pip install -r requirements/dev.txt`，`python manage.py runserver`；前端 `pnpm dev`。
- 依赖锁：`uv pip compile --generate-hashes` 输出 `requirements/{base,dev,prod}.txt`；前端 `pnpm-lock.yaml` + `npm-audit-resolver`。
- 镜像：第一版 `apps/api/Dockerfile` + `apps/web/Dockerfile`，distroless 基底；非 root；只读 FS；多架构 `linux/amd64,linux/arm64`。
- Service：`deploy/database/postgres/` 与 `deploy/cache/redis/` 独立 Compose。
- 不主动 build 应用镜像；仅在用户明确要求或 CI 触发时构建。

## 质量门禁

- 后端：`ruff` + `mypy --strict-equality` + `bandit` + `semgrep` + `pip-audit` + `safety`。
- 前端：`eslint` + `tsc --noEmit` + `@next/eslint` + `pnpm audit`。
- 镜像：`trivy fs` + `trivy image`；SBOM (`syft`) + 镜像签名 (`cosign`)。
- 单元 + 集成 + 安全测试。

## 不引入的技术（含理由与触发条件）

- `worker` 异步队列：第一版无异步任务；触发：心跳吞吐过大或邮件队列必要。
- 对象存储：第一版无文件上传；触发：合规导出格式需大文件附件。
- Elasticsearch：第一版无搜索；触发：审计跨年检索吞吐瓶颈。
- 多租户中间件：第一版只有厂商内部使用；触发：客户自助门户上线。

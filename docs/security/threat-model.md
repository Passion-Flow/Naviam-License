# 威胁模型 (STRIDE)

本文件覆盖 License 平台的威胁建模。每条威胁必须有：影响、缓解、检测、复核频率。

## 资产

| 资产 | 价值 | 影响等级 |
|---|---|---|
| 厂商 Ed25519 签发私钥 | 决定 License 真实性，泄漏后全网 License 不可信 | 致命 |
| 默认管理员账号 | 所有签发权 | 致命 |
| Postgres 主库（含审计哈希链） | 业务真相来源 | 高 |
| 审计哈希链 | 法律/合规证据 | 高 |
| Cloud ID / Activation Code 协议 | 校验逻辑核心 | 高 |
| 客户产品实例 SDK 嵌入的公钥 | 公开值；不可被替换为攻击者公钥 | 中（公开） |
| 客户实例公钥指纹与硬件指纹 | 用于绑定，泄漏后可用于伪造请求 | 中 |
| Console Session Cookie | 短期凭证 | 中 |

## 信任边界

- 浏览器 ↔ Web。
- Web ↔ API。
- API ↔ Postgres / Redis。
- Console ↔ 产品实例（在线模式 mTLS）。
- SDK ↔ 客户文件系统（离线模式）。

## STRIDE

### S - 仿冒（Spoofing）

| 威胁 | 缓解 | 检测 | 复核 |
|---|---|---|---|
| 攻击者伪装成厂商管理员登录 Console | 强密码 + Argon2id + 强制 2FA + 登录限速 + 锁定 + Session SameSite=Strict | `accounts_login_attempt` 异常激增、地理位置突变 | 周 |
| 攻击者伪装成产品实例向 Console 心跳 | mTLS + 公钥指纹绑定 + 心跳签名 | 心跳来源 IP、客户端公钥与历史不符 | 月 |
| 攻击者伪装 Console 给产品下发假 License | SDK 内置厂商公钥；任何非该公钥签名的 License 拒绝 | SDK 启动日志 / 校验失败计数 | 月 |
| 钓鱼站点骗取管理员凭证 | 仅在确定域 + HSTS + 物理 2FA Token（可选 V2） | 浏览器证书警告 / DNS 异常 | 季 |

### T - 篡改（Tampering）

| 威胁 | 缓解 | 检测 | 复核 |
|---|---|---|---|
| 攻击者修改 License 文件 | Ed25519 签名 + 规范化字节 + SDK 验签 | SDK 验签失败计数 | 月 |
| 攻击者修改审计记录 | 哈希链 + 服务端签名 | 链校验脚本 / 启动时全链 hash | 周 |
| 攻击者修改 SDK 公钥 | SDK 包发布签名 + 客户构建产物校验 | 客户构建报告 | 季 |
| 攻击者修改客户实例时钟以绕过过期 | SDK 双源时间（系统 + 构建时间）+ 在线模式时钟核对 | 时钟回拨日志 | 月 |
| 攻击者注入恶意 Activation Code | 解码失败 / 校验位失败 / 公钥指纹不匹配 即拒绝 | 解码失败计数 | 月 |

### R - 抵赖（Repudiation）

| 威胁 | 缓解 | 检测 | 复核 |
|---|---|---|---|
| 管理员事后否认签发某 License | 哈希链 + 服务端签名 + 操作 IP/UA | 审计导出可证伪 | 季 |
| 客户事后否认收到 Activation Code | Console 可显示通知 outbox 与签收回执 | outbox 状态 | 月 |
| 系统事后否认撤销 | 审计记录 + 通知 outbox + 撤销 webhook 收据 | outbox 状态 | 月 |

### I - 信息泄漏（Information disclosure）

| 威胁 | 缓解 | 检测 | 复核 |
|---|---|---|---|
| 私钥从加密文件被读出 | 加密文件 + passphrase + 启动时一次性解密到内存 + 进程边界保护 + 文件权限 0400 | 文件访问审计 | 周 |
| 私钥从内存被 dump | 仅签发进程持有；非签发进程不可读；生产禁用 ptrace | 异常进程白名单 | 季 |
| 后端日志打印密钥 / 密码 / token | 日志过滤 + 敏感字段红黑名单 + 单元测试断言 | 日志扫描器 | 周 |
| 错误响应泄漏 stack trace | 全站统一错误 + traceid；生产 DEBUG=False | E2E 错误页测试 | 月 |
| 数据库直接访问 | 仅服务网络可达 + scram-sha-256 + 强密码 | Postgres 连接审计 | 月 |
| 备份/导出包泄漏 | 加密压缩 + 客户合规存储 | 导出包接收方校验 | 季 |

### D - 拒绝服务（Denial of service）

| 威胁 | 缓解 | 检测 | 复核 |
|---|---|---|---|
| 暴力登录把账户锁死 | 锁定按 (用户 + IP) 维度，避免单 IP 锁全账户；管理员可一键解锁 | 锁定计数突增告警 | 周 |
| 大量 Cloud ID 解码消耗 CPU | 限速 + 解码时间预算 + 校验位先行（早期失败） | API 响应时间 | 月 |
| 心跳风暴 | 心跳限速 + 客户端最小间隔约束 | metrics | 月 |
| 邮件 / Webhook 通道被打爆 | 重试退避 + 上限 + 备用通道 | outbox 失败率 | 月 |

### E - 权限提升（Elevation of privilege）

| 威胁 | 缓解 | 检测 | 复核 |
|---|---|---|---|
| 攻击者通过 SQL 注入提权 | Django ORM + 参数化；禁止字符串拼 SQL；bandit + semgrep | 静态扫描 + 入站参数审计 | 月 |
| 攻击者通过 SSRF 触达内部网络 | 出站白名单 + Webhook 必须显式协议 + DNS 解析约束 + 拒绝 169.254/10/172/192.168 | 出站日志 | 月 |
| 攻击者越权读取其他客户 License | 视图层显式按 actor + 资源所属过滤；服务函数二次校验 | 越权回归测试 | 月 |
| 攻击者通过反序列化提权 | 禁用 Pickle；JSON / DRF 序列化器 | 静态扫描 | 季 |
| 攻击者通过 CSRF 触发签发 | Django CSRF 中间件 + SameSite=Strict + Origin 校验 | E2E 测试 | 月 |
| 攻击者通过开放重定向钓鱼 | 重定向白名单 | E2E 测试 | 月 |

## 失败模式与默认行为

- 私钥不可加载 -> API 拒启动。
- Postgres 不可达 -> API 5xx；不降级。
- Redis 不可达 -> 限速 / 锁定 fail-closed。
- Webhook 通道不可达 -> 重试 + 退避；不影响主流程。
- 时钟异常 -> SDK 在容忍窗口内告警，超出窗口拒绝运行。

## 离线撤销限制（已知物理限制）

离线模式下，Console 撤销不能立即触达客户实例。该限制不可消除。缓解：

- 短有效期：默认建议 ≤ 90 天，触发频繁续期。
- C3 可选：高敏客户配置「必选在线检查」；产品端启动必须能联网完成最近一次心跳。
- 撤销通知：邮件 + Webhook + Console 显式告知撤销在产品端到期前不会立即生效。

## 复核机制

- 每季度回看 STRIDE 表，更新失效项。
- 每次新增模块或新增对外接口，必须在本表新增对应行。
- 重大事件（密钥泄漏 / 越权 / DoS）触发即时复核 + ADR。

# Roadmap

按 `docs/design/12-开发路径与模块化规范.md` 推进。本文件用作短期/中期路线图的滚动摘要。

## 第一版（V1）目标

- 厂商可以在 Console 完成签发、续期、撤销，全部进入哈希链审计。
- 同时支持在线 / 离线激活；离线优先且自洽。
- 默认管理员强制改密 + 2FA；审计可签名导出。
- Postgres + Redis；仅 api + web 两个镜像。
- 产品端 Python SDK 可离线校验。

## 第二版（V2）方向（不进入第一版）

- 多产品支持上线第二、第三个产品（架构已就绪）。
- KMS / HSM 替换 A 方案。
- 客户自助门户（独立网络域）。
- 多角色 RBAC（厂商内部协同）。
- 用量统计与续费提醒邮件。

## 持续优化

- 安全：每季度一次依赖扫描全量回补；外部渗透测试一年一次。
- 性能：审计跨年检索观测；必要时引入分区或 OLAP。
- 体验：Cloud ID / Activation Code 误粘贴提示与校验位反馈。

## 不做（保护边界）

- 不做按 feature 颗粒度签发。
- 不做财务计费。
- 不做客户 SaaS。
- 不做开放注册。
